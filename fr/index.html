Intégration d'agent · Exécuteur local

Use Signal Horse as your local trading workspace for AI agents

Signal Horse est un espace de trading local et une API HTTP pour les agents IA et les opérateurs. Il permet de connecter Binance, OKX, Bybit, Bitget, Gate et d'autres plateformes en un clic pour le trading automatisé par IA, le trading manuel et le trading par lots sur plusieurs comptes. Charts, soldes, positions et exécution des ordres restent sur votre machine ; les identifiants de plateforme restent en local. OpenClaw est un client pris en charge, mais l'utilisation ne s'y limite pas.

Ce qui est installé sur votre machine

Signal Horse est installé sur la même machine que celle sur laquelle vous tradez. L'installation télécharge le programme, crée des fichiers de données locaux et peut activer le démarrage automatique pour que votre outil IA se connecte toujours au même service local.

Programme principal Signal Horse

C'est le programme local installé sur votre ordinateur. Il fournit la page de trading, la connexion locale et les actions de trading.

Paramètres et enregistrements locaux

Vos paramètres et autres données locales restent sur cette machine plutôt que dans un compte web distant.

Démarrage automatique en arrière-plan

Après l'installation, Signal Horse peut démarrer automatiquement en arrière-plan, vous n'avez plus besoin de le rouvrir manuellement à chaque fois.

Exchange key entry

Pour connecter une plateforme, renseignez l'API Key et le Secret. OKX et Bitget nécessitent également une phrase de passe.

Détails d'installation

Si vous installez vous-même, utilisez l'installateur standard. Pour préparer un serveur ou un workflow IA, utilisez la commande d'installation en une ligne.

Windows

Portable ZIP for the local browser-based trading UI. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260522.014131Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract signal_horse_windows_portable.zip
2. Open the extracted folder
3. Run Start Signal Executor.cmd
4. Your browser opens the local UI

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions.

macOS

Application de barre de menus qui installe Signal Horse localement et permet d'ouvrir la page de trading ou de redémarrer le service local en un clic.

Download Signal Horse for macOS (.dmg) Télécharger le DMG Intel macOS 13+ · Apple Silicon / Intel

Pour la plupart des utilisateurs macOS, utilisez l'application packagée :

1. Télécharger et ouvrir le package macOS Signal Horse (.dmg)
2. Faire glisser l'application téléchargée dans /Applications
3. Ouvrir l'application et utiliser l'icône de la barre de menus pour ouvrir l'interface locale dans le navigateur
4. Utiliser le menu pour redémarrer le service local ou activer le démarrage en arrière-plan

Ce DMG peut être ouvert normalement dans Finder, mais n'est pas encore notarisé par Apple. La première fois, macOS peut vous demander de faire un clic droit sur l'application et de choisir Ouvrir.

Pour un déploiement en ligne de commande ou une installation sans surveillance, vous pouvez toujours utiliser la commande d'installation en une ligne :

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

L'installation en ligne de commande configure le même service local Signal Horse. L'application est simplement un moyen plus pratique de le gérer depuis le bureau.

Linux (Ubuntu / Debian)

Exécutez une commande et Linux téléchargera Signal Horse, l'installera, configurera le démarrage automatique et ouvrira le service local sur le port 38182 :

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

Une fois terminé, vérifiez que Signal Horse fonctionne :

curl http://127.0.0.1:38182/health

Pour accéder à la page Signal Horse depuis un autre appareil, autorisez le port TCP 38182 dans le pare-feu Linux ou le groupe de sécurité, puis visitez :

http://<server-ip>:38182/

Par défaut, Signal Horse utilise le port 38182 sous Linux. L'installateur tente de configurer le démarrage automatique pour l'utilisateur actuel ; sans root, il s'installe dans le répertoire utilisateur.

FAQ

Quick answers to the questions most users ask before they start.

Cela installe-t-il Signal Horse sur mon ordinateur ?

Oui. L'installateur télécharge Signal Horse sur votre ordinateur et peut le maintenir en cours d'exécution localement en arrière-plan pour votre outil IA ou page de trading.

Mes clés API d'exchange passent-elles par votre serveur?

Non. Vos clés API restent sur votre propre machine. Signal Horse les lit localement et envoie les requêtes directement à la plateforme.

Which port does Signal Horse use?

Par défaut, Signal Horse utilise le port 38182. Sur la même machine, ouvrez http://127.0.0.1:38182/ . Pour y accéder depuis un autre appareil Linux, ouvrez d'abord ce port dans le pare-feu.

Comment rester en sécurité lors de l'activation du trading IA ?

Commencer par le mode démo, conserver la confirmation manuelle pour les vrais ordres et définir des limites de levier et de risque avant de laisser l'IA placer des ordres réels.
