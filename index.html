Use Signal Horse as your local trading workspace for AI agents

Signal Horse is a local trading workspace and HTTP API for AI agents and operators. It lets you connect Binance, OKX, Bybit, Bitget, Gate, and other supported exchanges in one click for AI automated trading, while also supporting manual trading and batch trading across multiple accounts. Charts, balances, positions, and spot or swap order execution stay on one machine, and exchange credentials remain on the local host. OpenClaw is one supported client, but the flow is not limited to OpenClaw.

What gets installed on your machine

Signal Horse is installed on the same machine where you trade. Setup downloads the program, creates local data files, and can enable automatic startup so your AI tool always connects to the same local service.

Signal Horse core program

This is the local program installed on your computer. It provides the trading page, local connection, and trading actions.

Local settings and records

Your settings and other local data stay on this machine instead of in a remote web account.

Automatic background startup

After installation, Signal Horse can start automatically in the background, so you do not need to reopen it manually every time.

Exchange key entry

To connect an exchange, fill in API Key and Secret. OKX and Bitget also require a passphrase.

Installation details

If you are installing it yourself, use the normal installer. If you are preparing a server or AI workflow, use the one-line install command.

Windows

Portable ZIP for the local browser-based trading UI. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260522.014131Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract signal_horse_windows_portable.zip
2. Open the extracted folder
3. Run Start Signal Executor.cmd
4. Your browser opens the local UI

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions.

macOS

Menu bar app that installs Signal Horse locally and lets you open the trading page or restart the local service with one click.

Download Signal Horse for macOS (.dmg) Download Intel DMG macOS 13+ · Apple Silicon / Intel

For most macOS desktop users, use the packaged app:

1. Download and open the Signal Horse macOS package (.dmg)
2. Drag the downloaded app into /Applications
3. Open the app and use the menu bar icon to open the local UI in your browser
4. Use the menu to restart the local service or turn on background startup

This DMG can be opened normally in Finder, but it is not yet Apple notarized. The first time, macOS may still ask you to right click the app and choose Open once.

If you need command-line deployment or unattended setup, you can still use the one-line install command:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

The command-line install sets up the same local Signal Horse service. The app is simply a more convenient desktop way to manage it.

Linux (Ubuntu / Debian)

Run one command and Linux will download Signal Horse, install it, set it to start automatically, and open the local service on port 38182:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

After it finishes, confirm that Signal Horse is running:

curl http://127.0.0.1:38182/health

If you want to open the Signal Horse page from another device, allow TCP port 38182 on the Linux firewall or security group, then visit:

http://<server-ip>:38182/

By default, Signal Horse uses port 38182 on Linux. The installer tries to set it to start automatically for the current user, and if you do not use root it installs under your user directory.

FAQ

Quick answers to the questions most users ask before they start.

Will this install Signal Horse on my computer?

Yes. The installer downloads Signal Horse to your computer and can keep it running locally in the background for your AI tool or trading page.

Do my exchange API keys go through your server?

No. Your API keys stay on your own machine. Signal Horse reads them locally and sends requests from your machine directly to the exchange.

Which port does Signal Horse use?

By default, Signal Horse uses port 38182. On the same machine, open http://127.0.0.1:38182/ . If you want to access it from another Linux device, open that port in the firewall first.

How should I stay safe when enabling AI trading?

Start with demo mode, keep manual confirmation for real orders, and set leverage and risk limits before letting AI place live trades.
