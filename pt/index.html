Integração de agente · Executor local

Use o Signal Horse como seu espaço de trabalho de trading local para agentes de IA

O Signal Horse é um espaço de trabalho de trading local e uma API HTTP para agentes de IA e operadores. Permite conectar Binance, OKX, Bybit, Bitget, Gate e outras exchanges suportadas com um clique para trading automatizado por IA, trading manual e trading em lote em várias contas. Charts, saldos, posições e execução de ordens ficam em uma máquina local; as credenciais da exchange permanecem no host local. OpenClaw é um cliente suportado, mas o fluxo não se limita ao OpenClaw.

O que é instalado na sua máquina

O Signal Horse é instalado na mesma máquina onde você faz trading. A instalação baixa o programa, cria arquivos de dados locais e pode habilitar a inicialização automática para que sua ferramenta de IA sempre se conecte ao mesmo serviço local.

Programa principal do Signal Horse

É o programa local instalado no seu computador. Fornece a página de trading, conexão local e ações de trading.

Configurações e registros locais

Suas configurações e outros dados locais ficam nesta máquina em vez de em uma conta web remota.

Inicialização automática em segundo plano

Após a instalação, o Signal Horse pode iniciar automaticamente em segundo plano, sem precisar abri-lo manualmente toda vez.

Inserção de chaves da exchange

Para conectar uma exchange, preencha a API Key e o Secret. OKX e Bitget também exigem uma passphrase.

Detalhes da instalação

Se você está instalando, use o instalador padrão. Para preparar um servidor ou fluxo de trabalho de IA, use o comando de instalação em uma linha.

Windows

Portable ZIP for the local browser-based trading UI. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260518.064959Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract signal_horse_windows_portable.zip
2. Open the extracted folder
3. Run Start Signal Executor.cmd
4. Your browser opens the local UI

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions.

macOS

App da barra de menus que instala o Signal Horse localmente e permite abrir a página de trading ou reiniciar o serviço local com um clique.

Baixar Signal Horse para macOS (.dmg) Baixar DMG Intel macOS 13+ · Apple Silicon / Intel

Para a maioria dos usuários macOS desktop, use o app empacotado:

1. Baixe e abra o pacote macOS do Signal Horse (.dmg)
2. Arraste o app baixado para /Applications
3. Abra o app e use o ícone da barra de menus para abrir a interface local no navegador
4. Use o menu para reiniciar o serviço local ou ativar o início em segundo plano

Este DMG pode ser aberto normalmente no Finder, mas ainda não tem notarização da Apple. Na primeira vez, o macOS pode pedir que você clique com o botão direito no app e escolha Abrir uma vez.

Se precisar de implantação por linha de comando ou configuração autônoma, ainda pode usar o comando de instalação em uma linha:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

A instalação por linha de comando configura o mesmo serviço local do Signal Horse. O app é simplesmente uma forma mais conveniente de gerenciá-lo pelo desktop.

Linux (Ubuntu / Debian)

Execute um comando e o Linux baixará o Signal Horse, instalará, configurará o início automático e abrirá o serviço local na porta 38182:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

Após concluir, confirme que o Signal Horse está em execução:

curl http://127.0.0.1:38182/health

Para abrir a página do Signal Horse em outro dispositivo, permita a porta TCP 38182 no firewall Linux ou grupo de segurança e acesse:

http://<server-ip>:38182/

Por padrão, o Signal Horse usa a porta 38182 no Linux. O instalador tenta configurar o início automático para o usuário atual; sem root, é instalado no diretório do usuário.

Fluxo recomendado para a primeira execução

A primeira execução mais simples: instale o Signal Horse, adicione suas chaves API da exchange, verifique saldos e posições e decida entre trading demo ou real.

1
Instalar o Signal Horse localmente

Instale-o na mesma máquina onde sua ferramenta de IA ou fluxo de trabalho de trading é executado.

2
Adicionar chaves API da exchange

Preencha a API Key e o Secret. OKX e Bitget também exigem uma passphrase.

3
Verificar dados antes de operar

Antes de colocar ordens reais, confirme que saldos, posições e histórico de ordens podem ser lidos corretamente.

4
Escolher modo de execução

Use primeiro o modo demo ou testnet. Mude para trading real apenas após confirmar a conta e as permissões corretas.

Perguntas frequentes

Respostas rápidas às perguntas mais comuns antes de começar.

Isso instalará o Signal Horse no meu computador?

Sim. O instalador baixa o Signal Horse para o seu computador e pode mantê-lo em execução localmente em segundo plano para sua ferramenta de IA ou página de trading.

Minhas chaves API da exchange passam pelo seu servidor?

Não. Suas chaves API ficam na sua própria máquina. O Signal Horse as lê localmente e envia solicitações da sua máquina diretamente para a exchange.

Qual porta o Signal Horse usa?

Por padrão, o Signal Horse usa a porta 38182. Na mesma máquina, abra http://127.0.0.1:38182/ . Para acessar de outro dispositivo Linux, abra essa porta no firewall primeiro.

Como me manter seguro ao ativar o trading com IA?

Comece com o modo demo, mantenha confirmação manual para ordens reais e defina limites de alavancagem e risco antes de permitir que a IA coloque operações reais.
