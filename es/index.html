Integración de agente · Ejecutor local

Usa Signal Horse como espacio de trabajo de trading local para agentes de IA

Signal Horse es un espacio de trabajo de trading local y una API HTTP para agentes de IA y operadores. Permite conectar Binance, OKX, Bybit, Bitget, Gate y otros exchanges compatibles con un clic para trading automatizado por IA, trading manual y trading por lotes en múltiples cuentas. Los charts, saldos, posiciones y ejecución de órdenes permanecen en una máquina local; las credenciales del exchange quedan en el host local. OpenClaw es un cliente compatible, pero el flujo no se limita a OpenClaw.

Qué se instala en tu máquina

Signal Horse se instala en la misma máquina donde realizas trading. La instalación descarga el programa, crea archivos de datos locales y puede habilitar el inicio automático para que tu herramienta de IA siempre se conecte al mismo servicio local.

Programa principal de Signal Horse

Es el programa local instalado en tu ordenador. Proporciona la página de trading, la conexión local y las acciones de trading.

Configuraciones y registros locales

Tu configuración y otros datos locales permanecen en esta máquina en lugar de en una cuenta web remota.

Inicio automático en segundo plano

Tras la instalación, Signal Horse puede iniciarse automáticamente en segundo plano, sin necesidad de reabrirlo manualmente cada vez.

Entrada de claves del exchange

Para conectar un exchange, rellena la API Key y el Secret. OKX y Bitget también requieren una passphrase.

Detalles de instalación

Si instalas tú mismo, usa el instalador estándar. Para preparar un servidor o flujo de trabajo de IA, usa el comando de instalación en una línea.

Windows

Portable ZIP for the local browser-based trading UI. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260511.053243Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract signal_horse_windows_portable.zip
2. Open the extracted folder
3. Run Start Signal Executor.cmd
4. Your browser opens the local UI

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions.

macOS

App de la barra de menús que instala Signal Horse localmente y permite abrir la página de trading o reiniciar el servicio local con un clic.

Descargar Signal Horse para macOS (.dmg) Descargar DMG Intel macOS 13+ · Apple Silicon / Intel

Para la mayoría de usuarios de escritorio macOS, usa la app empaquetada:

1. Descarga y abre el paquete macOS de Signal Horse (.dmg)
2. Arrastra la app descargada a /Applications
3. Abre la app y usa el icono de la barra de menús para abrir la interfaz local en el navegador
4. Usa el menú para reiniciar el servicio local o activar el inicio en segundo plano

Este DMG puede abrirse normalmente en Finder, pero aún no está notariado por Apple. La primera vez, macOS puede pedirte que hagas clic derecho en la app y elijas Abrir una vez.

Si necesitas despliegue por línea de comandos o instalación desatendida, puedes usar el comando de instalación en una línea:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

La instalación por línea de comandos configura el mismo servicio local de Signal Horse. La app es simplemente una forma más cómoda de gestionarlo desde el escritorio.

Linux (Ubuntu / Debian)

Ejecuta un comando y Linux descargará Signal Horse, lo instalará, lo configurará para inicio automático y abrirá el servicio local en el puerto 38182:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

Al terminar, confirma que Signal Horse está en ejecución:

curl http://127.0.0.1:38182/health

Para abrir la página de Signal Horse desde otro dispositivo, permite el puerto TCP 38182 en el firewall de Linux o grupo de seguridad, luego visita:

http://<server-ip>:38182/

Por defecto, Signal Horse usa el puerto 38182 en Linux. El instalador intenta configurar el inicio automático para el usuario actual; sin root se instala en el directorio del usuario.

Flujo recomendado para el primer inicio

El primer inicio más sencillo: instala Signal Horse, añade tus claves API del exchange, comprueba saldos y posiciones y decide si usar trading demo o real.

1
Instalar Signal Horse localmente

Instálalo en la misma máquina donde se ejecuta tu herramienta de IA o flujo de trabajo de trading.

2
Añadir claves API del exchange

Rellena la API Key y el Secret. OKX y Bitget también requieren una passphrase.

3
Comprobar datos antes de operar

Antes de colocar órdenes reales, confirma que los saldos, posiciones e historial de órdenes se pueden leer correctamente.

4
Elegir modo de ejecución

Usa primero el modo demo o testnet. Cambia a trading real solo después de confirmar la cuenta y los permisos correctos.

Preguntas frecuentes

Respuestas rápidas a las preguntas más frecuentes antes de comenzar.

¿Esto instalará Signal Horse en mi ordenador?

Sí. El instalador descarga Signal Horse en tu ordenador y puede mantenerlo en ejecución localmente en segundo plano para tu herramienta de IA o página de trading.

¿Mis claves API del exchange pasan por vuestro servidor?

No. Tus claves API permanecen en tu propia máquina. Signal Horse las lee localmente y envía solicitudes desde tu máquina directamente al exchange.

¿Qué puerto usa Signal Horse?

Por defecto, Signal Horse usa el puerto 38182. En la misma máquina, abre http://127.0.0.1:38182/ . Para acceder desde otro dispositivo Linux, abre primero ese puerto en el firewall.

¿Cómo mantenerme seguro al activar el trading con IA?

Comienza con el modo demo, mantén la confirmación manual para órdenes reales y establece límites de apalancamiento y riesgo antes de dejar que la IA coloque operaciones reales.

Signal Horse puede conectarse desde OpenClaw, Claude Code, Codex, Hermes Agents y otras plataformas de agentes para acceso centralizado a múltiples exchanges de criptomonedas y gestión unificada.

OpenClaw

Los flujos de trabajo de agentes locales pueden llamar a Signal Horse para acceso a exchanges, ejecución de órdenes y automatización del trading.

Claude Code

Usa Claude Code como copiloto de trading local para investigación, ejecución manual y flujos de trabajo automatizados.

Codex

Codex puede usar Signal Horse como puerta de enlace de trading local para investigación cripto, ejecución de órdenes y automatización del trading con IA.

Hermes Agents

Hermes Agents y otros runtimes multiagente pueden unificar las operaciones en exchanges de criptomonedas a través de Signal Horse.
