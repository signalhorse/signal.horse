Agent-Integration · Lokaler Executor

Use Signal Horse as your local trading workspace for AI agents

Signal Horse ist ein lokaler Trading-Arbeitsbereich und eine HTTP-API für KI-Agenten und Betreiber. Er ermöglicht die Verbindung mit Binance, OKX, Bybit, Bitget, Gate und anderen unterstützten Börsen per Klick für KI-Automatisierung, manuelle Orders und Batch-Trading über mehrere Konten. Charts, Salden, Positionen und Orderausführung bleiben auf einem Gerät; Zugangsdaten verbleiben lokal. OpenClaw ist ein unterstützter Client, aber der Ablauf ist nicht darauf beschränkt.

Was auf Ihrem Rechner installiert wird

Signal Horse wird auf demselben Rechner installiert, auf dem Sie handeln. Das Setup lädt das Programm herunter, erstellt lokale Datendateien und kann den Autostart aktivieren, sodass Ihr KI-Tool immer mit demselben lokalen Service verbunden ist.

Signal Horse Kernprogramm

Dies ist das lokale Programm auf Ihrem Rechner. Es stellt die Handelsseite, die lokale Verbindung und Handelsaktionen bereit.

Lokale Einstellungen und Aufzeichnungen

Ihre Einstellungen und andere lokale Daten bleiben auf diesem Gerät statt in einem externen Web-Account.

Installationsdetails

Wenn Sie selbst installieren, verwenden Sie den normalen Installer. Wenn Sie einen Server oder KI-Workflow vorbereiten, verwenden Sie den Einzeilen-Installationsbefehl.

Windows

Portable ZIP for the local browser-based trading UI. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260518.124948Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract signal_horse_windows_portable.zip
2. Open the extracted folder
3. Run Start Signal Executor.cmd
4. Your browser opens the local UI

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions.

macOS

Menüleisten-App, die Signal Horse lokal installiert und die Handelsseite oder den lokalen Service per Klick öffnet bzw. neu startet.

Download Signal Horse for macOS (.dmg) Intel DMG herunterladen macOS 13+ · Apple Silicon / Intel

Für die meisten macOS-Desktop-Nutzer: gepackte App verwenden:

1. Signal Horse macOS-Paket (.dmg) herunterladen und öffnen
2. Heruntergeladene App in /Applications ziehen
3. App öffnen und Menüleistemsymbol verwenden, um die lokale Oberfläche im Browser zu öffnen

Dieses DMG kann normal im Finder geöffnet werden, ist aber noch nicht von Apple notarisiert. Beim ersten Mal könnte macOS Sie auffordern, die App mit Rechtsklick zu öffnen und einmal „Öffnen" zu wählen.

Für Kommandozeilen-Deployment oder unbeaufsichtigtes Setup steht weiterhin der Einzeilen-Installationsbefehl zur Verfügung:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

Die Kommandozeileninstallation richtet denselben lokalen Signal Horse Service ein. Die App ist ein komfortablerer Desktop-Weg, ihn zu verwalten.

Linux (Ubuntu / Debian)

Einen Befehl ausführen und Linux lädt Signal Horse herunter, installiert es, richtet den Autostart ein und öffnet den lokalen Service auf Port 38182:

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

Nach Abschluss bestätigen, dass Signal Horse läuft:

curl http://127.0.0.1:38182/health

Um die Signal Horse Seite von einem anderen Gerät aufzurufen, TCP-Port 38182 in der Linux-Firewall freigeben, dann aufrufen:

http://<server-ip>:38182/

Standardmäßig verwendet Signal Horse Port 38182 unter Linux. Der Installer versucht, den Autostart für den aktuellen Benutzer einzurichten; ohne root wird es im Benutzerverzeichnis installiert.

Nach der Installation verfügbare lokale Adressen

GET  http://127.0.0.1:38182/health
GET  http://127.0.0.1:38182/positions
GET  http://127.0.0.1:38182/balances
POST http://127.0.0.1:38182/place-order
POST http://127.0.0.1:38182/cancel-order
POST http://127.0.0.1:38182/set-tpsl
POST http://127.0.0.1:38182/cancel-tpsl
POST http://127.0.0.1:38182/close-all

Auf demselben Gerät kann Ihr KI-Tool weiterhin die obige lokale Adresse verwenden. Unter Linux können Sie nach Öffnen der Firewall auch von einem anderen Gerät auf die Webseite zugreifen.

FAQ

Quick answers to the questions most users ask before they start.

Wird Signal Horse auf meinem Computer installiert?

Ja. Der Installer lädt Signal Horse auf Ihren Computer herunter und hält es lokal im Hintergrund für Ihr KI-Tool oder Ihre Handelsseite am Laufen.

Werden meine Börsen-API-Schlüssel über Ihren Server geleitet?

Nein. Ihre API-Schlüssel bleiben auf Ihrem eigenen Gerät. Signal Horse liest sie lokal und sendet Anfragen direkt an die Börse.

Which port does Signal Horse use?

Standardmäßig verwendet Signal Horse Port 38182. Öffnen Sie auf demselben Gerät http://127.0.0.1:38182/ . Für den Zugriff von einem anderen Linux-Gerät zuerst diesen Port in der Firewall öffnen.

Wie bleibe ich beim Aktivieren des KI-Handels sicher?

Mit Demo-Modus beginnen, manuelle Bestätigung für echte Orders beibehalten und Leverage- sowie Risikolimits setzen, bevor dem KI Live-Trades erlaubt werden.
