将 Signal Horse 用作 AI 智能体的本地交易工作台

Signal Horse 是面向 AI 智能体和操作员的本地交易工作台与 HTTP API。它支持一键连接 Binance、OKX、Bybit、Bitget、Gate 等受支持交易所，用于 AI 自动化交易，也支持手动交易以及跨多个账户的批量交易。图表、余额、持仓，以及现货或合约下单都在同一台机器上完成，交易所凭证也保留在本地主机。OpenClaw 是其中一个受支持的客户端，但整体流程并不局限于 OpenClaw。

安装说明

如果你是自己安装，请使用常规安装方式；如果你在准备服务器或 AI 工作流，请使用一行安装命令。

Windows

面向本地浏览器交易界面的便携 ZIP 包。不需要管理员权限，不写注册表，也不会自动启动。

下载 Windows 便携 ZIP 包 Windows x64 · 1.0.0+20260521.021641Z

将 ZIP 解压到任意用户可写目录，然后运行本地启动器：

1. 下载并解压 signal_horse_windows_portable.zip
2. 打开解压后的文件夹
3. 运行 Start Signal Executor.cmd
4. 浏览器会自动打开本地界面

如果你更喜欢用脚本安装到 LocalAppData，请使用这个 PowerShell 辅助脚本：

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

本地界面地址：

http://127.0.0.1:38182/

这个 Windows 包始终只在用户目录内运行。不需要管理员权限，不写开机启动注册表项，只会提示你有新版本可用。

macOS

会在本地安装 Signal Horse 的菜单栏应用，并让你一键打开交易页面或重启本地服务。

下载 Signal Horse for macOS (.dmg) 下载 Intel DMG macOS 13+ · Apple Silicon / Intel

对大多数 macOS 桌面用户来说，直接使用打包好的应用即可：

1. 下载并打开 Signal Horse macOS 安装包 (.dmg)
2. 将下载的应用拖入 /Applications
3. 打开应用，然后使用菜单栏图标在浏览器中打开本地界面
4. 通过菜单重启本地服务，或打开后台自动启动

这个 DMG 可以在 Finder 中正常打开，但还没有完成 Apple notarization。第一次打开时，macOS 仍可能要求你右键应用并选择一次“打开”。

如果你需要命令行部署或无人值守安装，也可以继续使用一行安装命令：

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

命令行安装配置的是同一套本地 Signal Horse 服务；图形应用只是更方便的桌面管理方式。

Linux (Ubuntu / Debian)

运行一条命令后，Linux 会自动下载并安装 Signal Horse、设置自动启动，并在 38182 端口启动本地服务：

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

完成后，请确认 Signal Horse 正在运行：

curl http://127.0.0.1:38182/health

如果你想从另一台设备打开 Signal Horse 页面，请先在 Linux 防火墙或安全组中放行 TCP 38182 端口，然后访问：

http://<server-ip>:38182/

默认情况下，Signal Horse 在 Linux 上使用 38182 端口。安装器会尝试为当前用户启用自动启动；如果你不是用 root 安装，它会安装到你的用户目录下。

推荐的首次使用流程

最简单的首次使用流程是：先安装 Signal Horse，添加交易所 API 密钥，检查余额和持仓，再决定使用 demo 还是真实交易。

1
在本地安装 Signal Horse

请安装在运行 AI 工具或交易工作流的同一台机器上。

2
添加交易所 API 密钥

填写 API Key 和 Secret 即可。OKX 和 Bitget 还需要 passphrase。

3
交易前先检查数据

在下真实订单前，先确认余额、持仓和订单历史都能正确读取。

4
选择执行模式

先使用 demo 或 testnet。只有在确认账户和权限都正确之后，再切换到真实交易。

常见问题

这里汇总了大多数用户开始前最常问的问题。

这会把 Signal Horse 安装到我的电脑上吗？

会。安装程序会把 Signal Horse 下载到你的电脑上，并可让它在后台本地运行，供你的 AI 工具或交易页面使用。

我的交易所 API 密钥会经过你们的服务器吗？

不会。你的 API 密钥只保留在你自己的机器上。Signal Horse 会在本地读取它们，并由你的机器直接向交易所发出请求。

Signal Horse 使用哪个端口？

默认使用 38182 端口。在同一台机器上打开 http://127.0.0.1:38182/ 。如果要从另一台 Linux 设备访问，请先在防火墙中放行该端口。

启用 AI 交易时，怎样做更安全？

先从 demo 模式开始，真实下单保留人工确认，并在让 AI 发起真实交易前设置好杠杆和风险限制。

OpenClaw

Claude Code

Codex

Codex 可以把 Signal Horse 用作本地交易网关，用于加密货币研究、订单执行和 AI 交易自动化。

Hermes Agents

Hermes Agents 和其他多智能体运行时也可以通过 Signal Horse 统一处理加密货币交易所操作。
