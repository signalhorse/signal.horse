將 Signal Horse 作為 AI 智能體的本地交易工作台

Signal Horse 是面向 AI 智能體與操作員的本地交易工作台與 HTTP API。它支援一鍵連接 Binance、OKX、Bybit、Bitget、Gate 等受支援交易所，用於 AI 自動化交易，也支援手動交易以及跨多個帳戶的批量交易。圖表、餘額、持倉，以及現貨或合約下單都在同一台機器上完成，交易所憑證也保留在本地主機。OpenClaw 是其中一個受支援的客戶端，但整體流程並不侷限於 OpenClaw。

你的機器上會安裝哪些內容

Signal Horse 會安裝在你實際交易的那台機器上。安裝流程會下載程式、建立本地資料檔案，並可啟用自動啟動，讓你的 AI 工具始終連接到同一個本地服務。

Signal Horse 核心程式

這是安裝在你電腦上的本地程式，提供交易頁面、本地連接以及交易操作。

本地設定與記錄

你的設定和其他本地資料都會保留在這台機器上，而不是存到遠端網頁帳戶裡。

背景自動啟動

安裝完成後，Signal Horse 可以在背景自動啟動，你不必每次都手動重新打開。

交易所密鑰填寫

連接交易所時，填寫 API Key 和 Secret 即可。OKX 和 Bitget 還需要 passphrase。

安裝說明

如果你是自己安裝，請使用常規安裝方式；如果你在準備伺服器或 AI 工作流程，請使用一行安裝命令。

Windows

面向本地瀏覽器交易介面的便攜 ZIP 包。不需要管理員權限，不寫入註冊表，也不會自動啟動。

下載 Windows 便攜 ZIP 包 Windows x64 · 1.0.0+20260521.021641Z

將 ZIP 解壓到任何使用者可寫目錄，然後運行本地啟動器：

1. 下載並解壓 signal_horse_windows_portable.zip
2. 打開解壓後的資料夾
3. 運行 Start Signal Executor.cmd
4. 瀏覽器會自動打開本地介面

如果你更喜歡用腳本安裝到 LocalAppData，請使用這個 PowerShell 輔助腳本：

Invoke-WebRequest -Uri https://signal.horse/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

本地介面地址：

http://127.0.0.1:38182/

這個 Windows 包始終只在使用者目錄內運行。不需要管理員權限，不寫入開機啟動註冊表項，只會提示你有新版本可用。

macOS

會在本地安裝 Signal Horse 的選單列應用，並讓你一鍵打開交易頁面或重啟本地服務。

下載 Signal Horse for macOS (.dmg) 下載 Intel DMG macOS 13+ · Apple Silicon / Intel

對大多數 macOS 桌面使用者來說，直接使用打包好的應用即可：

1. 下載並打開 Signal Horse macOS 安裝包 (.dmg)
2. 將下載的應用拖入 /Applications
3. 打開應用，然後使用選單列圖示在瀏覽器中打開本地介面
4. 透過選單重啟本地服務，或開啟背景自動啟動

這個 DMG 可以在 Finder 中正常打開，但還沒有完成 Apple notarization。第一次打開時，macOS 仍可能要求你右鍵應用並選擇一次「打開」。

如果你需要命令列部署或無人值守安裝，也可以繼續使用一行安裝命令：

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

命令列安裝配置的是同一套本地 Signal Horse 服務；圖形應用只是更方便的桌面管理方式。

Linux (Ubuntu / Debian)

運行一條命令後，Linux 會自動下載並安裝 Signal Horse、設定自動啟動，並在 38182 連接埠啟動本地服務：

curl -fsSL https://signal.horse/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

完成後，請確認 Signal Horse 正在運行：

curl http://127.0.0.1:38182/health

如果你想從另一台裝置打開 Signal Horse 頁面，請先在 Linux 防火牆或安全組中放行 TCP 38182 連接埠，然後訪問：

http://<server-ip>:38182/

預設情況下，Signal Horse 在 Linux 上使用 38182 連接埠。安裝器會嘗試為當前使用者啟用自動啟動；如果你不是用 root 安裝，它會安裝到你的使用者目錄下。

建議的首次使用流程

最簡單的首次使用流程是：先安裝 Signal Horse，添加交易所 API 密鑰，檢查餘額和持倉，再決定使用 demo 還是真實交易。

1
在本地安裝 Signal Horse

請安裝在運行 AI 工具或交易工作流程的同一台機器上。

2
添加交易所 API 密鑰

填寫 API Key 和 Secret 即可。OKX 和 Bitget 還需要 passphrase。

3
交易前先檢查資料

在下真實訂單前，先確認餘額、持倉和訂單歷史都能正確讀取。

4
選擇執行模式

先使用 demo 或 testnet。只有在確認帳戶和權限都正確之後，再切換到真實交易。

常見問題

這裡整理了大多數使用者開始前最常問的問題。

這會把 Signal Horse 安裝到我的電腦上嗎？

會。安裝程式會把 Signal Horse 下載到你的電腦上，並可讓它在背景本地運行，供你的 AI 工具或交易頁面使用。

我的交易所 API 密鑰會經過你們的伺服器嗎？

不會。你的 API 密鑰只保留在你自己的機器上。Signal Horse 會在本地讀取它們，並由你的機器直接向交易所發出請求。

Signal Horse 使用哪個連接埠？

預設使用 38182 連接埠。在同一台機器上打開 http://127.0.0.1:38182/ 。如果要從另一台 Linux 裝置訪問，請先在防火牆中放行該連接埠。

啟用 AI 交易時，怎樣做更安全？

先從 demo 模式開始，真實下單保留人工確認，並在讓 AI 發起真實交易前設定好槓桿和風險限制。

支援的軟體平台

Signal Horse 可以從 OpenClaw、Claude Code、Codex、Hermes Agents 等智能體平台接入，一站式連接多個加密貨幣交易所並統一管理。

OpenClaw

本地智能體工作流程可以調用 Signal Horse 完成交易所接入、訂單執行和交易自動化。

Claude Code

可將 Claude Code 用作本地交易副駕駛，用於研究、手動執行和自動化工作流程。

Codex

Codex 可以把 Signal Horse 用作本地交易網關，用於加密貨幣研究、訂單執行和 AI 交易自動化。

Hermes Agents

Hermes Agents 和其他多智能體運行時也可以透過 Signal Horse 統一處理加密貨幣交易所操作。
